Estrategias Usadas

Las herramientas usadas fueron los semáforos e hilos

La estrategia de sincronización utilizada en este código se basa en el uso de semáforos para coordinar la comunicación y la interacción entre los hilos. Los semáforos son una herramienta fundamental para la sincronización en programación concurrente, ya que permiten controlar el acceso a recursos compartidos y coordinar la ejecución de diferentes partes del programa

En particular, este código utiliza dos semáforos: wakeWorker y requestClient. Explicaré cómo se utilizan y qué función desempeñan en la coordinación de la ejecución de hilos

wakeWorker - Semáforo para despertar a los trabajadores:

Este semáforo se utiliza para despertar a los trabajadores cuando hay una solicitud de un cliente. Inicialmente, su valor es 0, lo que significa que los trabajadores estarán dormidos hasta que el semáforo se libere. Cuando un cliente genera una solicitud (requestClient.release()), se libera el semáforo wakeWorker (wakeWorker.release()), lo que despierta a un trabajador para que atienda la solicitud

requestClient - Semáforo para indicar la solicitud de un cliente:

Este semáforo se utiliza para indicar que hay una solicitud de un cliente. Cuando un cliente tiene una nueva solicitud, se libera el semáforo (requestClient.release()). En la función del jefe (Boss), se espera en este semáforo (requestClient.acquire()) hasta que un cliente genera una solicitud. Una vez que se libera, el jefe despierta a un trabajador para que atienda la solicitud

La estrategia es la siguiente:

El jefe (Boss) espera en el semáforo requestClient hasta que un cliente genere una solicitud. Cuando un cliente genera una solicitud, libera el semáforo requestClient. El jefe despierta a un trabajador liberando el semáforo wakeWorker. Un trabajador, que ha sido despertado por el jefe, atiende la solicitud y luego se va a descansar.Esta estrategia asegura que los trabajadores estén dormidos hasta que haya una solicitud y se despierten solo cuando haya algo que hacer. Los semáforos garantizan que la sincronización ocurra de manera adecuada, evitando condiciones de carrera y asegurando que los hilos se coordinen de manera efectiva en la atención de las solicitudes de los clientes.
